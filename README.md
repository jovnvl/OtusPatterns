# OtusPrototype
Класс на самом верхнем уровне, от него идет вся иерархия
1-класс: Composition - художественное, музыкальное, зодчество и прочее искусство
	Id	      - идентификационный номер 
	Title	  - название искусства
	Duration  - направление в искусстве, из справочника:
		   класс: Id	- учетный номер
			 Name	    - название

следующий класс наследуется от Composition
2-класс: MusicComposition - музыкальное искусство
	Mark	  - уровень оценки
	BeginDate - дата возникновения
	Genre	  - жанр в искусстве, из справочника:
		   класс: Id	- учетный номер
			 Name	    - название
следующий класс наследуется от MusicComposition
3-класс: PopularComposition - популярная музыка
	Author	  - автор музыкального произведения
	CreateDate- дата создания музыкального произведения
	Style	  - стиль музыкального произведения, из справочника:
		   класс: Id	- учетный номер
			 Name	    - название

Выводы по преимуществам и недостаткам у каждого из интерфейсов: IMyCloneable и ICloneable:

при реализации интерфейса ICloneable метод Clone() возвращает тип object,
поэтому результат требует приведения к искомому классу,
при реализации интерфейса IMyCloneable метод DeepClone() возвращает тип искомого класса.
с другой стороны, в некоторых случаях более удобно использовать общий метод Clone(),
так как может не потребоваться ссылаться в методе на конкретные поля и свойства класса,
а использовать, например общий метод MemberwiseClone(), которому не нужно знать об этом.

поскольку нет требований для реализации метода Clone() должен он возвращать поверностную или полную копию экземпляра класса,
то можно поведение метода реализовать по своему усмотрению и как рекомендует компания Microsoft
интерфейс ICloneable можно не реализовывать в общедоступных API, так как вызывающие методы Clone() не могут зависеть от метода,
выполняющего предсказуемую операцию клонирования.
